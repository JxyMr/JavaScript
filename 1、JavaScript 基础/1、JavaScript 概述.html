Web 程序员的“内力”和“外功”

  1. JavaScript 是 Web 程序员的核心“内力”（原生JS）
  2. jQuery、React、AngularJS、Sea.js等框架技术是建立在JavaScript基础上的“外功” （大多数对自己无要求的前端开发，只会用，只知其然不知其所以然）

JavaScript 概述

		JavaScript 是一种基于“对象”，“解释型”脚本语言。
		JavaScript 是一种对让网页“动起来”的语言。

		对象：生活中各种各样的事物。如：人、飞机、笔记本都有自己不同的数据和功能。
			JavaScript中用来写代码的各种各样不同的元素、组成部分称为对象。

		解释型：代码通过以行为单位，逐行解释并执行。若有bug需要执行后才以发现。
		编译型：把代码全部翻译成机器码，若有bug会立即提示并且报错。必须全部调试完毕最终才能执行。

		它是运行在本地（客户端）浏览器环境下的脚本语言。
		它的代码是明文的，被缓存在本地浏览器的历史记录中。

		java、c++、.net这些语言一般需要运行时环境（Runtime），而JavaScript的运行环境是浏览器。

JavaScript 能做什么？

  1. 动态改变网页中的样式
  2. 与用户的操作进行互动（用户交互）
  3. 验证表单输入内容
  4. 改变浏览器本身的表现

Java 和 JavaScript

  • Java（Sun） 和 JavaScript（Netscape网景）是两种不同的语言，无论在概念还是设计上
  • Java多用来开发大型的独立桌面应用程序，是一门真正的面向对象语言。
  • JavaScript用来开发网页应用程序，最初诞生的目的仅仅是网页脚本，验证表单。

JavaScript和ECMAScript

		JavaScript 诞生于1995年，由Netscape网景公司的“布兰登·艾奇（Brendan Eich，1961~）”
		其被尊为“JavaScript 之父”

		JavaScript 的前身叫 LiveScript，后来 Netscape 和 Sun 达成开发联盟，为了占上Java的风
		头，临时改名为 JavaScript，所以实际上JavaScript和Java压根没有半毛钱关系。

		IE 的JScript，网景的JavaScript，ScriptEase中的CEnvi
		ECMA（European Computer Manufactures Association）-欧洲计算机制造商协会
		交给其进行标准化工作，第一个标准化版本称为 ECMA-262，而符合这个标准的语言称为 ECMAScript。

		JavaScript和ECMAScript通常被人看作相同的意思，实际上JavaScript所包含的内容比
		ECMAScript要多太多，JavaScript包含3个方面：

		* 核心语言（ECMAScript）
		* 文档对象模型（DOM）
		* 浏览器对象模型（BOM）
